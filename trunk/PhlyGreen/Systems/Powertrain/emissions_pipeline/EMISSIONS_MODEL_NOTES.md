# GT emissions surrogate — feasibility assessment & roadmap

*Assessment only (no code). Goal: a state-dependent GT emissions surrogate (NOx/CO/UHC, later
soot) coupled to PhlyGreen's universal GT map, giving emissions = f(altitude, speed, power/rpm).*

## 1. What the thesis materials actually provide

Both theses build the **same physics core** — *combustor inlet state → emission indices* —
implemented as Cantera Chemical Reactor Networks (CRN) and calibrated to **ICAO LTO data on the
CFM56**:

| | Pietrosanto (`WIP/TESI_PIETROSANTO`) | Bassis (`WIP/Codici Tesi Fabrizio Bassis`) |
|---|---|---|
| Species | **NOx, CO, UHC** | **soot** |
| Chemistry | Cantera CRN, **Luche** kerosene surrogate + an **evaporation** model | Cantera CRN, **ethylene** surrogate, bypass‑PSR network + **omnisoot** |
| Calibration | tuned to ICAO LTO (CFM56/LEAP/TRENT) | **differential evolution** (24‑core) → `Calibrato.txt` |
| Surrogate | NN per output (5 NNs) | NN → `surrogate_CFM56.pkl` |
| Kernel signature | `evap_model_ottimizzato.run_crn_evap_mode("GRID", params)` with `(T3, P3, FAR, ṁ_air, dP/P)` | CRN fn `(T_in, p_in, ṁ_air, FAR, dP/P)` → `EI_soot` |

**Pietrosanto already built the whole chain** we want, for CFM56:
`OpenAP envelope grid (Mach, alt, power-code) → pyCycle → combustor state (T3_K, P3_Pa, FAR,
ṁ_air3) → CRN → (EINOx, EICO, EIUHC)`. The resulting **1996‑point dataset exists as CSV**
(`…/CFM/pycycle/…` and `…/SCIENCE_PLOTS/pycycle_process/crn_pycycle_final_patchato.csv`). Bassis's
soot dataset is `dati_sintetici3_CFM56.csv` (422 pts).

The decisive structural fact: **the mapping is two-stage and separable** —
`(alt, speed, power) --pyCycle--> combustor state --CRN--> EI`. The CRN is (to first order)
*engine-agnostic given the combustor inlet state* + a small set of combustor-geometry calibration
params; the engine-specific part lives in the pyCycle cycle.

## 2. The integration target (favourable)

PhlyGreen's GT map is generated by an **in-package pyCycle deck**,
`trunk/PhlyGreen/Systems/Powertrain/data/Single_spool_GT.py`, which **already instantiates a
`pyc.Combustor`** (inlet = `comp.Fl_O` → `T3, P3`; `burner.Wfuel`, `FAR`). Today only *thermal
efficiency* is written to `GT_Universal_Map.csv` and fitted (RBF + StandardScaler) by
`train_gas_turbine_surrogate.py` into `GT_Engine_Model_Complete.pkl`, loaded at runtime by
`gas_turbine_surrogate.py`. So the **combustor state needed to drive emissions is already computed**
— it just isn't exported yet. The existing surrogate-packaging convention (CSV → RBF → `.pkl`) and
the `Systems/Powertrain/efficiency.py` `ResponseSurfaceEfficiency` interface are the natural homes
for the EI surrogate; `ClimateImpact` is the consumer (it currently uses the Filippone NOx
correlation and fixed EIs for CO/soot/etc. — see the recently-removed GasTurb path).

## 3. Environment status (gates reproduction)

Checked in the `phlygreen` env: **`pycycle` 4.4, `openmdao` 3.44, `scikit-learn` 1.7 — present.**
**`cantera`, `omnisoot`, `openap`, `openpyxl` — absent.** Consequences:
- pyCycle decks (incl. a PW127 cycle) **can be run now**.
- Re-running the CRN/soot needs a **Cantera (+omnisoot) install** — a Phase‑0 setup task.
- The **existing CFM56 EI/soot CSVs let the surrogate + PhlyGreen integration be built today
  without Cantera.**
- The PW127 emissions **Excel needs `openpyxl`** to read.

### 3a. PW127 calibration data — verified (FOCA/EASA emission-factor DB)
The Excel is the FOCA "Emission factors" DB. The single PW127 entry is **`PW127G`** (Type `T`
turboprop, **Power 2750 kW**, source **EASA 2024**, "Sample IV D5"). It gives per‑LTO‑mode
**EI in g/kg fuel + fuel flow** — exactly the ICAO‑style calibration targets the CRN needs:

| mode (≈%F00) | FF [kg/s] | NOx | VOC (UHC) | CO | nvPMF (soot) |
|---|---|---|---|---|---|
| Taxi (idle) | 0.04 | 5.0 | **0.0** | 20.0 | 1.2 |
| App (30) | 0.08 | 10.0 | **0.0** | 4.0 | 0.25 |
| Cl (85) | 0.14 | 16.0 | **0.0** | 2.0 | 0.55 |
| To (100) | 0.16 | 19.0 | **0.0** | 2.0 | 0.3 |

**Findings:** (i) NOx, CO and **soot (nvPMF)** are present and physically sensible (NOx rises with
power, CO falls, soot peaks at idle/take-off) → good calibration targets. (ii) **VOC/UHC is 0.0 at
every mode** → *there is no PW127 calibration target for unburned hydrocarbons.* So the realistic
PW127 gaseous-species deliverable is **NOx + CO** (UHC can be predicted by the CRN but not
calibrated/validated for PW127). (iii) Only the 4 ground LTO points exist (as for CFM56); the
envelope extension comes from pyCycle, same as Pietrosanto. (iv) `Times in Mode` for turboprops:
To 0.5 / Cl 2.5 / App 4.5 / Taxi 26 min.

## 4. Recommendations (you asked me to choose)

### 4a. Engine sequencing → **CFM56 first (as scaffolding), PW127 as the production target**
Do **not** start on PW127: that couples three hard unknowns at once (a new pyCycle cycle + a CRN
re-calibration to PW127 ICAO + the PhlyGreen wiring). Instead exploit the separable two-stage
structure:
- **Stand up the entire surrogate + GT-map integration on CFM56 using the existing data** (no
  Cantera). This de-risks and proves the PhlyGreen-side machinery end-to-end.
- **Then retarget to PW127** — the engine that actually matches the GT map — by running a PW127
  pyCycle cycle and **re-calibrating the CRN to the PW127 ICAO points** (your Excel). This is the
  long pole but, by then, only *one* unknown is left.

CRN calibration is **combustor-specific and not transferable** CFM56→PW127, so PW127 numbers are
only credible after that recalibration. CFM56 results are scaffolding, not the deliverable.

### 4b. Species → **NOx + CO + UHC first; soot as a later add-on**
Gaseous species (Pietrosanto) are Cantera-only, one kerosene mechanism, and have ready data — ship
them first. **Soot (Bassis) is a separate Phase**: it needs `omnisoot`, an **ethylene** surrogate
and DE calibration, i.e. a *different* mechanism/network than the gaseous model. **Unifying soot and
gaseous species onto one mechanism/network is itself a research sub-task** (don't assume the two
CRNs merge for free).

### 4c. 3-shaft PW127 cycle → **defer; first re-tune the simplified deck to PW127, upgrade only if needed**
pyCycle fully supports multi-spool (the current deck already has HP+LP shafts; a true PW127 adds an
LP centrifugal spool + LP turbine on a third shaft, with extra maps, spool-speed balances and
**convergence tuning** — Pietrosanto already hit Newton NaNs at take-off). **But emissions depend on
combustor inlet `(T3, P3, FAR, ṁ)`, which are set mainly by OPR, the T4/ITT schedule and core mass
flow** — a 2-spool deck *matched to PW127's design point* reproduces these well; the marginal benefit
of a literal 3-shaft for *combustor state* is likely small next to the CRN's own calibration
uncertainty. Recommendation: **(B.1)** re-tune the existing single/2‑spool deck to PW127 design
(OPR, ITT, ṁ, power) and **validate combustor state against the Excel/ICAO**; **(B.2)** build the
full 3‑shaft only if the off-design combustor-state error is shown to matter. Don't gold-plate the
cycle before the surrogate proves it needs it.

## 5. Phased roadmap

- **Phase 0 — Reproducibility env (small).** Add `cantera`, `omnisoot`, `openpyxl`, `openap` to a
  dedicated env; read the PW127 Excel; smoke-test `Single_spool_GT.py` (pyCycle) and one Pietrosanto
  CRN point. Pin versions; these are heavy/optional deps (gate them like CoolProp/pycycle already are
  in PhlyGreen).
- **Phase 1 — EI surrogate from existing CFM56 data (no Cantera). ✅ DONE.** See
  `emissions_pipeline/ (prototype) `. Fit `(T3, P3, FAR, ṁ) → (EINOx, EICO, EIUHC)` from
  `crn_pycycle_final_patchato.csv` (1995 pts). Held-out test **R² = 0.9998 / 1.0000 / 0.9990**
  (NOx/CO/UHC). Portable artifact `emission_surrogate_CFM56.pkl` (plain StandardScaler + Rbf/RF, loads
  standalone) + `parity_emissions_CFM56.png`. Script is reusable for the PW127 dataset.
- **Phase 2 — Wire into PhlyGreen GT map (CFM56 scaffold). ✅ DONE.** See
  `WIP/phase2_gtmap_integration/`. Baked the **direct** `EI(alt_ft, Mach, PC)` map (held-out
  R²=0.9999/1.0/0.9992); added the package loader **`Systems/Powertrain/emissions_surrogate.py`**
  (`EmissionSurrogate`, mirrors `GasTurbineResponseSurface`, gated + domain-clipped); added the
  **opt-in `ClimateImpact.mission_emissions_from_surrogate()`** that integrates
  `EI(alt,Mach,power) × fuel-flow` over the mission for NOx/CO/UHC (default off — Filippone
  unchanged). Demo + EI-vs-time plot reproduce the right physics. Scaffold note: surrogate NOx is
  ~2× Filippone because CFM56 (turbofan) EINOx > turboprop — fixed by Phase 3 PW127 recalibration.
  Deferred to Phase 5: ship a PW127 artifact as default + make `ClimateImpact` use it automatically.
  The deck-export of `T3/P3/FAR/Wfuel` (the *state-based* online path) is folded into Phase 3 with
  the PW127 pyCycle run.
- **Phase 3 — Retarget to PW127 (production). 🔄 FOUNDATION DONE; recalibration scoped.** See
  `emissions_pipeline/`. (B.1) ✅ re-tuned the 2‑shaft deck to PW127 (OPR 14.7, rated 3290 hp/2453 kW)
  — take-off/climb fuel flow match the Excel to +0.1 %/+3.5 %; ✅ generated the combustor-state
  envelope (100/100 converged, **FAR 0.008–0.017 — much leaner than the CFM56 box 0.019–0.024**); ✅
  Cantera 3.2 installed and the **CRN runs (~3.4 s/pt)**. **Recalibration gap quantified** at the 4
  LTO modes: the CFM56-calibrated CRN over-predicts high-power NOx ~55–62 % and collapses at PW127's
  lean idle/approach. Of the two coupled fixes: **(i) deck part-power
  fuel-flow correction — ✅ DONE** (calibrated gas-generator power schedule `G(f)` matches all 4 LTO
  fuel flows to ±1.8 %; corrected idle/approach CO now matches ICAO: App 3.6 vs 4, Taxi 26.9 vs 20);
  **(ii) CRN combustor re-tune — partial.** ARPZ (primary-zone air fraction) is the dominant lever
  (`chi_mixer_scale` is saturated at these states). With the corrected states + a richer PW127 PZ
  (ARPZ≈0.24): **CO recalibrates to ICAO at all 4 LTO modes**; **NOx does not** — the simplified
  equal-split CRN's NOx-vs-power is too peaky to match PW127's flat 5→19 g/kg (Cl ~2× over, idle
  unreachable; rms ≈67 %), a model-structure limit (PW127 stages fuel / runs a hot pilot the CRN
  lacks). NOx path options: accept rough CRN NOx / certification-anchored NOx / add a staged-pilot PZ.
  Separately verified the **GT thermal-efficiency map already matches PW127 certification within ~5 %
  in the flight envelope** → fuel burn and emissions are consistent in flight; no GT-map revision
  needed (the part-power fix was a combustor-*state* fix for this emissions deck). (B.2) 3‑shaft
  deferred.
- **Phase 5 — Productionize. ✅ DONE.** The PW127 surrogate is shipped in the package:
  `Systems/Powertrain/data/PW127_Emission_Map.csv` (the EI dataset) + `Emission_Model_PW127.pkl`
  (the default artifact) + `train_emission_surrogate.py` (offline trainer, sklearn-only) +
  `emissions_surrogate.py` (runtime loader). `ClimateImpact` gained `EINOx_model='Surrogate'`
  (auto-loads the artifact, adds NOx/CO/UHC; Filippone stays default → golden-safe). Tests:
  `tests/regression/test_emission_surrogate.py`. Example: `examples/24_gt_emissions_surrogate.py`.
  Docs updated. The heavy CRN+pyCycle *generation* pipeline stays in `emissions_pipeline/` (gated by
  Cantera/pyCycle), documented as the provenance of the shipped CSV.
- **Phase 4 — Soot. Deferred (groundwork below).** See §7.

## 6. Key risks
- **CRN calibration is engine/combustor-specific** → PW127 needs its own ICAO recalibration.
- **Heavy/optional deps** (Cantera, omnisoot) + **pyCycle convergence** at idle/take-off (documented
  NaNs) → offline-only, gated, with robust point-filtering (Pietrosanto's `filtro_punti_fisici`).
- **Soot maturity & mechanism unification** (ethylene vs kerosene) is an open research item.
- **Research-grade code**: manual workflow, hard-coded paths — needs a real refactor to enter the
  package.
- **Compute**: CRN dataset generation is slow (parallel batch) but **offline**; the shipped artifact
  (surrogate) is fast, exactly like the existing GT map.

## 7. How to validate this assessment (read-only / cheap)
1. Fit an RBF on `crn_pycycle_final_patchato.csv` (`(T3,P3,FAR,ṁ)→EI`) with the installed sklearn and
   check parity — proves Phase 1 without Cantera.
2. Run `Single_spool_GT.py` (pyCycle is installed) at a few `(alt, Mach, PC)` and confirm it converges
   and exposes `burner` inlet `T3/P3/FAR/ṁ` — proves the Phase‑2 coupling point exists.
3. Install `openpyxl`, read the PW127 Excel, and sanity-check its ICAO LTO EIs against the CFM56
   dataset magnitudes — scopes the Phase‑3 recalibration.

## 8. Bottom line
**Feasible, and most of the hard pieces already exist.** The pyCycle→CRN→EI chain is built (CFM56)
and PhlyGreen's GT deck already computes the combustor state needed to drive it. The work was
**integration + retargeting**, not invention. **Done:** Phases 1–3 + 5 — a PW127 emission surrogate
(NOx/CO/UHC) shipped in the package, certification-anchored NOx, used via `einox_model='Surrogate'`.

## 9. Soot — groundwork for a future phase (deferred)

Soot (non-volatile PM) is intentionally **out of scope for now** but the path is clear and the data
exists. This section records what is in hand and what remains, so it can be picked up cleanly.

### What we have
- **A second CRN, for soot** — Bassis's thesis (`WIP/Codici Tesi Fabrizio Bassis/`): a Cantera CRN
  with an **ethylene surrogate fuel**, a bypass-PSR primary-zone network, **omnisoot** for soot
  formation/oxidation, calibrated to CFM56 by **differential evolution** (`Calibrato.txt`). Same
  input signature as the gaseous CRN — combustor inlet state `(T_in, p_in, ṁ_air, FAR, dP/P)` →
  `EI_soot`. A trained CFM56 soot surrogate (`surrogate_CFM56.pkl`) and synthetic data
  (`dati_sintetici3_CFM56.csv`, 422 pts) are included.
- **PW127 soot calibration targets** — the FOCA/EASA Excel gives **nvPMF (soot mass)** at the 4 LTO
  modes: **To 0.30, Cl 0.55, App 0.25, Taxi 1.20 g/kg** (idle highest — the classic rich-idle soot
  signature). So, unlike UHC, soot *does* have a PW127 anchor.
- **The exact same pipeline** we built for the gaseous species (the PW127 pyCycle deck +
  part-power-corrected combustor-state envelope `pw127_crn_inputs_corrected.csv`) feeds the soot CRN
  unchanged — soot just needs a different reactor network on the same states.

### What's to be done (estimated path)
1. **Env:** `pip install omnisoot` (Cantera already installed). The soot CRN is heavier than the
   gaseous one (particle dynamics) → slower per point.
2. **Run** Bassis's soot CRN over the PW127 corrected envelope → `EI_soot(alt, Mach, power)`.
3. **Calibrate to PW127 nvPMF** at the 4 LTO modes. Soot is *more* sensitive to primary-zone
   richness/temperature than NOx, so expect the same lever (`ARPZ` / the bypass split) to matter,
   and likely the same outcome shape: a physical CRN trend + a **certification anchor** to the nvPMF
   values (reuse the `k(PF)` anchoring from §3.7). Decide unify-vs-parallel: ideally one kerosene
   mechanism (Luche) with omnisoot on the *same* network as the gaseous species; Bassis used an
   **ethylene** surrogate, so unification is itself a sub-task (else keep two parallel CRNs).
4. **Add `EISOOT` as a 4th surrogate output.** The package format already supports arbitrary outputs
   — add `EISOOT` to `OUTPUTS` in `train_emission_surrogate.py`, regenerate the `.pkl`. The
   `EmissionSurrogate` loader and the `ClimateImpact` walk need no structural change (add `'soot'` to
   the `species` map).
5. **Wire into `ClimateImpact`:** today soot is a fixed EI × Wf (`EI_soot = 4e-5`); replace it with
   the surrogate value when `einox_model='Surrogate'` (the `rf_AIC`/contrail and `rf_soot` forcings
   then see a state-dependent soot). 
6. **Tests/example/docs:** mirror the gaseous ones; add an `EI_soot` parity test and a nvPMF anchor
   check.

### Risks specific to soot
- omnisoot + Cantera convergence over a wide envelope (lean/cool points may need clipping, as the
  gaseous CRN did).
- The ethylene-vs-kerosene mechanism mismatch (unification cost).
- Soot magnitudes span orders of magnitude and are noisier than NOx/CO → fit in log space (already
  the default for the small species), expect lower R² (UHC-like).
